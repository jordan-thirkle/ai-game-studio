# References

Curated best practices from the highest-quality repos in the AI agent, Three.js, web dev, and design ecosystems.

## Purpose

This directory organizes research so it's available when needed. It's reference material, not executable code. No skills, no automation, no new systems — just documented patterns.

## How to Use

### For Hermes
- Read these files when building new skills or improving the pipeline
- Reference specific patterns when implementing new features
- Draw from these when updating AGENTS.md or governance rules

### For Humans
- Quick reference for project conventions
- Source material for code reviews
- Training data for onboarding

## Categories

| Directory | Content | Key Sources |
|-----------|---------|-------------|
| `hermes/` | Agent patterns, skill lifecycle, memory architecture | awesome-hermes-agent, Hermes docs |
| `threejs/` | Three.js performance, architecture, gotchas | mrdoob/three.js, katopz/best-practices |
| `web-dev/` | TypeScript, testing, documentation standards | microsoft/TypeScript, tailwindcss |
| `design/` | Visual hierarchy, color, typography, components | Apple HIG, Material Design 3 |

## Maintenance

- Add new references as research is done
- Update existing files when better patterns are found
- Remove outdated patterns (mark as deprecated, don't delete)
- Keep files focused: one topic per file, clear title
