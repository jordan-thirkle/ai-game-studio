# AI Game Studio — Quality Framework

## Scoring System (100/100)

Every game and web app version is scored across 10 categories, each worth 10 points.

### Game Scoring (100 points)

| # | Category | Weight | What It Measures |
|---|----------|--------|-----------------|
| 1 | Art Direction | 10 | Color palette, theme cohesion, visual identity, aesthetic consistency |
| 2 | Hero/Player | 10 | Character design, animation quality, personality, movement feel |
| 3 | Obstacles/Enemies | 10 | Challenge design, variety, fairness, visual clarity |
| 4 | Rewards/Interactables | 10 | Collectible design, feedback quality, satisfaction, visual appeal |
| 5 | World/Environment | 10 | Scene composition, density, atmosphere, depth, storytelling |
| 6 | Materials/Textures | 10 | Surface quality, variation, realism, shader complexity |
| 7 | Lighting/Render | 10 | Shadows, ambient lighting, tone mapping, fog, post-processing |
| 8 | VFX/Motion | 10 | Particles, screen effects, game feel, juice, animation quality |
| 9 | UI/HUD | 10 | Clarity, responsiveness, mobile support, visual polish |
| 10 | Performance | 10 | FPS stability, load time, memory usage, bundle size, mobile |

### Web App Scoring (100 points)

| # | Category | Weight | What It Measures |
|---|----------|--------|-----------------|
| 1 | Visual Design | 10 | Color scheme, typography, spacing, visual hierarchy |
| 2 | User Experience | 10 | Navigation, flow, intuitiveness, error handling |
| 3 | Content Quality | 10 | Writing quality, documentation completeness, accuracy |
| 4 | Responsiveness | 10 | Mobile, tablet, desktop, touch targets, safe areas |
| 5 | Performance | 10 | Load time, Core Web Vitals, bundle size, caching |
| 6 | SEO/Meta | 10 | Title, description, OG images, structured data, robots |
| 7 | Accessibility | 10 | ARIA labels, keyboard nav, color contrast, screen readers |
| 8 | Features | 10 | Functionality completeness, edge cases, error states |
| 9 | Code Quality | 10 | TypeScript strictness, component architecture, reusability |
| 10 | Polish | 10 | Animations, transitions, loading states, empty states |

### Grade Scale

| Score | Grade | Meaning |
|-------|-------|---------|
| 90-100 | S | Production-ready, AAA quality |
| 80-89 | A | High quality, minor polish needed |
| 70-79 | B | Good, several areas need work |
| 60-69 | C | Functional, significant gaps |
| 50-59 | D | Prototype quality, major rework |
| 40-49 | F | Broken or incomplete |
| <40 | F- | Non-functional |

## Documentation Requirements

### Per Version (Every Iteration)

1. **Screenshot** — Full page / game view at 1080p
2. **Scorecard** — All 10 categories scored with notes
3. **Changelog** — What changed, why, what was fixed
4. **Issues Found** — Bugs, visual problems, performance issues
5. **Improvement Plan** — What's next, prioritized by impact
6. **Build Info** — Commit hash, build time, bundle size

### Per Game (Cumulative)

1. **Case Study** — Full narrative of the game's development
2. **Evolution Timeline** — All versions with scores and screenshots
3. **Technical Deep Dive** — Architecture, systems, shaders, audio
4. **Lessons Learned** — What worked, what failed, what was saved as skills
5. **X.com Thread** — Ready-to-post content for each major milestone

### Per Web App Version

1. **Visual Audit** — Full page screenshots of every route
2. **Performance Report** — Lighthouse scores, Core Web Vitals
3. **Feature Checklist** — What's working, what's not
4. **Content Audit** — All text reviewed for accuracy and quality

## Pipeline Enforcement

### Game Pipeline (7 Phases)

Each phase must be completed before the next begins. Each phase has entry/exit criteria:

- [ ] **Phase 1: Design Brief** — Player promise, target feeling, primary verbs, core loop contract
- [ ] **Phase 2: Gameplay Systems** — Architecture, mechanics, entities, input, camera, physics
- [ ] **Phase 3: Asset Generation** — Credential probe, generator loading, asset sourcing ledger
- [ ] **Phase 4: Graphics** — Lighting, materials, shaders, VFX, visual scorecard
- [ ] **Phase 5: UI/HUD** — Menus, overlays, responsive design, mobile controls
- [ ] **Phase 6: QA Testing** — Browser QA, performance, mobile, accessibility, screenshots
- [ ] **Phase 7: Scoring** — Full 100-point scorecard, improvement plan, documentation

### Web App Pipeline (Per Version)

- [ ] **Design Review** — Visual audit of all pages
- [ ] **Content Review** — All text proofread and accurate
- [ ] **Performance Check** — Lighthouse ≥90 on all metrics
- [ ] **SEO Audit** — Meta tags, OG images, structured data
- [ ] **Accessibility Check** — Keyboard nav, ARIA, contrast
- [ ] **Mobile Test** — Touch targets, responsive, safe areas
- [ ] **Cross-browser** — Chrome, Firefox, Safari, Edge
- [ ] **Deploy & Verify** — Build passes, live site works, no console errors

## Content Strategy

### X.com Posting Cadence

- **Every iteration:** Before/after screenshot + score comparison
- **Every game milestone:** Full thread with prompts, learnings, results
- **Weekly:** Summary of all improvements across projects
- **Monthly:** "State of the Studio" post with stats and roadmap

### Content Types

1. **Before/After** — Side-by-side screenshots showing improvement
2. **Score Cards** — Visual score breakdown with categories
3. **Prompt Transparency** — Exact prompts that produced results
4. **Failure Documentation** — What went wrong and how it was fixed
5. **Pipeline Insights** — How the 7-phase system works
6. **Self-Improvement** — Skills saved, learnings applied, compound growth
